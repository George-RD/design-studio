Design Studio

A 4-Agent Frontend Tool for Claude Code

Makes websites that look original, not like every other AI template. Works inside Claude Code with no extra setup.

View the Site Install

What's the Problem?

When you ask an AI to build a website, it usually makes the same thing over and over: a centered headline, a button, some cards in a grid. It looks fine, but it's boring. Every AI site starts to look the same.

The reason is code-anchoring bias. When the AI reads its own code before making changes, it just tweaks what's already there. It never stops to think "what should this really look like?"

How Design Studio Fixes This

Design Studio splits the work into four separate jobs. Each job is done by a different agent that only sees what it needs to see. This is how real design studios work — the art director never writes code, and the developer never makes creative choices.

Step 1 — Plan
The Planner
Takes your idea and turns it into a full plan. It picks a look and feel, lists what the page needs, and sets a creative direction.
Step 2 — Design
The Design Agent
Describes what the page should look like in plain words. It never sees any code — only screenshots and feedback. This stops it from copying old ideas.
Step 3 — Build
The Implementation Agent
Takes the design description and builds real code from it. It follows the design exactly — no shortcuts, no "I think this looks better."
Step 4 — Check
The Evaluator
Opens the page in a real browser, clicks around, takes screenshots, and scores the design. It checks for broken layouts, hard-to-read text, and boring patterns.

Then the loop repeats. The design gets better each time. When the scores are high enough, the tool ships the best version.

What Makes It Different

No more template websites

Most AI tools make the same centered-hero-with-card-grid layout. Design Studio pushes past that by separating the person who dreams up the design from the person who writes the code.

Real browser testing

The evaluator opens your page in Chrome, clicks buttons, scrolls, and checks for bugs. It doesn't just read the code — it uses the page like a real person would.

Zone-by-zone scoring

Instead of giving the whole page one score, it checks each section separately. A broken footer can't hide behind a pretty hero section.

Adversarial testing

Before scoring, the evaluator tries to break things. It checks for text that overflows, buttons that don't work, and layouts that fall apart on mobile. If something breaks, the score gets capped.

Install

You need Claude Code and Chrome with the claude-in-chrome MCP server running.

# Install from GitHub
claude plugin install https://github.com/George-RD/design-studio

# Or load it for one session
claude --plugin-dir ./design-studio

After installing, run /reload-plugins in Claude Code.

Quick Start

In Claude Code, type:

/design-studio:create a landing page for a small coffee shop

The tool will plan, design, build, and check your page. It runs several rounds to make the design better each time. All the work is saved in a folder called harness-output/.

How Scoring Works

The evaluator scores your page on four things:

  • Design Quality (2x weight) — Does it look good together?
  • Originality (2x weight) — Does it look like every other site?
  • Craft (1x weight) — Is the code clean and polished?
  • Functionality (1x weight) — Can people use it?

Design quality and originality matter most. That's because AI is already good at making things that work. The hard part is making things that stand out.

Commands

/design-studio:create <your prompt>

Run the full design loop. Give it a short description of what you want to build.

Who Made This

Based on research from Anthropic Labs about "harness design for long-running application development." The key insight: separating the creative eye from the code writer produces much better results than a single agent doing everything.
